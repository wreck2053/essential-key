Privacy policy

Your configuration stays on your device.

Effective June 23, 2026. This policy describes the current open-source application.

Summary

The Android app does not include advertising, analytics, crash-reporting SDKs, user accounts, or a developer-operated backend. The app stores configuration locally and communicates over the network only for user-configured actions and local Wireless Debugging setup. The project website uses optional Google Analytics only after a visitor accepts the analytics prompt.

Website analytics

If accepted, Google Analytics measures page views, approximate location, device and browser information, referral sources, and clicks on APK download links. The website sends a custom download_apk event when a visitor clicks the direct APK link. Declining prevents the analytics tag from loading. The preference is stored in the browser's local storage and can be reset by clearing site data.

Data stored locally

  • Selected hardware-key identifiers, including key code, scan code, input source, device descriptor, vendor ID, and product ID.
  • Configured gesture actions, URLs, app package names, labels, haptic strength, and recent action results.
  • An ADB identity used for local Wireless Debugging. Its exportable private key is encrypted at rest using a key held by Android Keystore.
  • Setup diagnostics such as timestamps, package-operation status, device model, Android version, and error details.

This information is stored in the app's private storage and is not automatically sent to the developer.

Accessibility service

The Accessibility service requests hardware key events and can perform user-selected Android global actions. The implementation does not process AccessibilityEvent screen content, inspect text displayed by other apps, or type on the user's behalf.

Installed apps

When displaying the Launch App selector, the app queries launchable applications installed on the device and shows their labels, package names, and icons. This list is processed on-device and is not sent to the developer.

Network communication

  • Wireless Debugging: local-device pairing and ADB communication used for the package setup explicitly initiated by the user.
  • HTTP actions: GET or POST requests sent to URLs configured by the user. Those destinations have their own privacy and security practices.
  • Open URL: URLs are handed to another installed Android app capable of opening them.

Permissions

The app may request Internet access, notification access for pairing-code entry, vibration, audio-setting controls, Notification Policy access for silent-mode actions, flashlight hardware access, and the Accessibility service described above.

Data deletion

Clearing the app's storage or uninstalling deletes its private configuration, diagnostics, and ADB identity. Uninstalling does not re-enable the Nothing packages changed during setup; restore Essential Space before uninstalling.

Children

The app is a device customization utility and is not directed to children.

Changes and contact

Policy changes will be published on this page and in the source repository. Privacy questions can be filed through GitHub Issues without including private credentials or logs.
