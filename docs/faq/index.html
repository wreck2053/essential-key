FAQ

Important questions before you install.

Is this an official Nothing app?

No. Essential Key Remapper is an independent community project and is not affiliated with, endorsed by, or supported by Nothing Technology Limited.

Why does setup use Wireless Debugging?

Nothing OS normally consumes the Essential Key through Essential Space and Essential Recorder. The local Wireless Debugging connection lets the app change only those package states for the current Android user after you approve pairing.

Does it require a computer?

No. Pairing and package setup happen on the phone. Split-screen or the setup notification can be used to enter Android's six-digit pairing code.

What does the Accessibility service do?

It requests filtered hardware key events so the selected key can be recognized while other apps are open. The implementation does not process accessibility screen content or type text.

What happens if I uninstall the app?

The disabled Nothing packages remain disabled. Restore Essential Space from the Setup page before uninstalling.

Can I reverse the setup?

Yes. The Restore Essential Space action re-enables the two packages. You may need to repeat Wireless Debugging pairing.

Where are settings stored?

Gesture configuration, mapped hardware-key identifiers, and recent action results are stored locally in the app's private storage. The ADB identity is also stored locally with its private key encrypted at rest using Android Keystore.

Does the app send analytics or personal data?

The current code does not include advertising, analytics, crash-reporting, accounts, or a developer-operated data server. User-configured HTTP actions send requests to the destination entered by the user.

How are updates installed?

Download signed APK updates from the project's GitHub Releases page. Android will accept an update only when it uses the same application ID and signing key as the installed build.

Where can I get support?

Use GitHub Issues. Do not post pairing codes, private URLs, access tokens, or complete diagnostic logs containing sensitive information.
